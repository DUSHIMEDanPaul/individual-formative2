# resto_kigali

A Flutter mobile application for discovering and managing restaurant listings in Kigali, Rwanda. Users can browse restaurants, create their own listings, search and filter by category, view locations on an interactive map, bookmark favourites, and rate services.

## Features

- **Authentication** — Sign up, login, logout with Firebase Auth + email verification enforcement
- **Listings CRUD** — Create, read, update, and delete restaurant listings stored in Cloud Firestore
- **Search & Filter** — Search listings by name and filter by category (Restaurant, Café, Fast Food, Bakery, Bar, Buffet, Diner, Picnic Area, Food Stall, Hotel)
- **Map Integration** — Interactive map view showing all listings with markers; detail page embeds a map with directions support (flutter_map + OpenStreetMap)
- **Bookmarks** — Save and manage favourite listings
- **Reviews & Ratings** — Rate and review restaurants with automatic average rating calculation
- **Settings** — View profile info, toggle notifications, logout

## Firebase Setup

1. Create a Firebase project at [console.firebase.google.com](https://console.firebase.google.com)
2. Enable **Authentication** → Email/Password sign-in method
3. Enable **Cloud Firestore** in production or test mode
4. Add your Android/iOS/Web app in Firebase project settings
5. Download `google-services.json` (Android) or `GoogleService-Info.plist` (iOS) and place in the respective platform folders
6. Firebase config is managed via `lib/firebase_options.dart` (generated by FlutterFire CLI)

## Firestore Collections

### `users` collection
Each document is keyed by the Firebase Auth UID.
```
users/{uid}
  ├── uid: String
  ├── email: String
  ├── displayName: String
  ├── phoneNumber: String?
  ├── notificationsEnabled: bool
  ├── bookmarkedListings: List<String>  (listing IDs)
  ├── createdAt: Timestamp
  └── updatedAt: Timestamp
```

### `listings` collection
Each document represents a restaurant/service listing.
```
listings/{listingId}
  ├── name: String
  ├── category: String
  ├── description: String
  ├── address: String
  ├── phoneNumber: String
  ├── website: String?
  ├── imageUrl: String
  ├── latitude: double
  ├── longitude: double
  ├── amenities: List<String>
  ├── rating: double
  ├── reviewCount: int
  ├── createdBy: String  (user UID)
  ├── createdAt: Timestamp
  └── updatedAt: Timestamp
```

### `listings/{listingId}/reviews` subcollection
```
reviews/{reviewId}
  ├── userId: String
  ├── userName: String
  ├── userEmail: String
  ├── rating: double
  ├── comment: String
  └── createdAt: Timestamp
```

## State Management — Provider

The app uses **Provider** (ChangeNotifier) for state management. No UI widget directly calls Firebase APIs — all Firestore and Auth operations flow through provider classes.

### Data Flow

```
Firestore / Firebase Auth
        ↓
  Service Layer (FirestoreService, LocationService)
        ↓
  State Layer (AuthProvider, ListingProvider)
        ↓
  UI Widgets (Consumer / Provider.of)
```

- **AuthProvider** — Manages authentication state (sign up, sign in, sign out, email verification), user profile data from Firestore, and notification preferences. Exposes `isLoading`, `errorMessage`, `user`, `isEmailVerified`.
- **ListingProvider** — Manages all listings CRUD operations, search, category filtering, bookmarks, and reviews. Exposes `listings`, `userListings`, `bookmarkedListings`, `reviews`, `isLoading`, `errorMessage`. After any create/update/delete, it automatically re-fetches data so all screens rebuild.

Loading, success, and error states are handled for all operations. The UI uses `Consumer<ListingProvider>` and `Consumer<AuthProvider>` to rebuild automatically when state changes.

## Navigation

The app uses a `BottomNavigationBar` with 5 tabs:
1. **Directory** — Browse all listings with search and category filter chips
2. **My Listings** — View/manage listings created by the current user
3. **Map View** — Interactive map showing all listings as markers
4. **Bookmarks** — Saved/favourite listings
5. **Settings** — Profile info, notification toggle, logout

Authentication flow: `LoginScreen` → `EmailVerificationScreen` (if not verified) → `HomeScreen`

## Getting Started

```bash
# Install dependencies
flutter pub get

# Run the app
flutter run
```

- Flutter SDK 3.x+
- Firebase project with Auth and Firestore enabled
